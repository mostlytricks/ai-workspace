# Dashboard Design System (DESIGN.dashboard.md)

> [!IMPORTANT]
> **This is the source-of-truth design system for the `gravity · mission control` dashboard**
> — the offline HTML at `.claude/dashboard/dashboard.html`, generated by
> `generate_dashboard.py` from `PROJECTS.md`. An agent restyling or regenerating the
> dashboard reads this file first, then edits the `TEMPLATE` in `generate_dashboard.py` to match.
>
> **Not** for reading-docs (`MISSION.html` etc. use the flat muted-teal `DESIGN.docs.md`) and
> **not** for slides/Outlook HTML. The dashboard is built for **scanning at a glance**; it is
> deliberately louder and more luminous than the doc theme. Two surfaces, two systems.

A premium, dark **glassmorphism** mission control with a cosmic identity: deep-navy canvas
under a twinkling starfield, an **orbital map** (gravity as the sun, tiers as rings, projects
as planets), ambient radial glows, semi-transparent blurred cards, luminous color-coded tiers,
and dashed "move-line" thresholds.

---

## 0. Strategy & invariants (read before editing)

- **Generated, not hand-written.** The HTML is emitted by `generate_dashboard.py`; never hand-edit `dashboard.html` (it's overwritten + git-ignored). Change the `TEMPLATE` string / color constants in the generator, then rerun `python .claude/dashboard/generate_dashboard.py`.
- **Offline / self-contained.** No CDN at view time (`file://` blocks fetch). Chart.js **and** the fonts are vendored under `vendor/`; data is baked into the page as inline JSON. Keep it that way — re-vendor assets rather than linking out.
- **Stdlib-only generator.** No pip/venv (workspace rule). Pure-Python string templating via token replacement (`__DATA_JSON__`, `__CARDS__`, `__GENERATED__`, `__TOTAL__`) — so CSS/JS braces stay literal.
- **Data source = `PROJECTS.md`.** One bar/slice/card per indexed project; the dashboard inherits any index drift. It does not scan disk.
- **Semantics drive color, not decoration.** Every hue means something (tier, or staleness). Don't add color that doesn't carry meaning.
- **Chrome is themeable; semantics are not.** The palette below is the **default theme (`aurora`)**. A top-right switcher swaps five themes via `[data-theme]` on `<html>` (persisted in `localStorage` under `dash-theme`). Themes only change *chrome* — bg/surface/ink, ambient glows, glass shadow, hover ring, chart axes/grid. **Tier-accent hues and the staleness heat scale stay constant across all themes** (they're meaning, not style) — so they're still set per-section in Python, not in a theme block.

## 0a. Themes

Five palettes, default `aurora`. Two light, three dark — the set deliberately spans beyond the original futuristic look. Each is a self-contained block of CSS custom properties: `:root, [data-theme="aurora"]` for the default, then `[data-theme="…"]` overrides. Charts read their chrome from CSS vars (`--chart-axis`, `--chart-grid`, `--bg` for the slice border) and are **destroyed + rebuilt on every switch** so they repaint against the active palette.

| Theme | Feel | Canvas |
|---|---|---|
| `aurora` *(default)* | deep-navy neon, futuristic | `#0A0E1A` |
| `daylight` | clean light, professional blue | `#F7F9FC` |
| `sandstone` | warm light, cream + terracotta | `#FBF6EF` |
| `forest` | calm dark green | `#0C1A14` |
| `slate` | neutral graphite, muted/corporate | `#16181D` |

**Adding/editing a theme:** add or edit one `[data-theme="x"]` block in the generator's CSS (mirror every var the `aurora` block defines — `--bg --surface --surface-hover --border --border-focus --ink --muted --focus-text --chip-bg --selection --h1-grad --shadow --shadow-hover --ring-a --ring-b --chart-axis --chart-grid --stars --body-bg`), add a `<button data-theme="x">` with a signature swatch to the `#themebar` in the header, and append the key to the `THEMES` array in the page script. Keep light-theme overlays subtle (low-alpha) so the glassmorphism still reads.

**`--stars`** is the starfield's opacity per theme (dark themes ≥.5, light themes ≤.35 so the twinkle stays ambient, never noisy). Star *color* is not themed separately — the canvas reads `--chart-axis` so stars always match the chrome.

## 1. Color tokens

The default-theme (`aurora`) `:root` palette (deep navy-black with luminous overlays):

| Token | Value | Purpose |
|---|---|---|
| `--bg` | `#0A0E1A` | page canvas (deep navy-black) |
| `--surface` | `rgba(17,24,43,0.7)` | glass card / panel backdrop |
| `--surface-hover` | `rgba(26,36,64,0.8)` | card hover background |
| `--border` | `rgba(255,255,255,0.08)` | default subtle borders |
| `--border-focus` | `rgba(0,242,254,0.4)` | hover/active border glow |
| `--ink` | `#F3F4F6` | titles, body text |
| `--muted` | `#9CA3AF` | labels, metadata, captions |

**Ambient background** (on `body`, `background-attachment:fixed`): three low-opacity radial
gradients — teal top-left, purple top-right, teal bottom-center — layered over `--bg` for depth.

```css
background:
  radial-gradient(900px 520px at 10% -8%, rgba(0,242,254,.10), transparent 60%),
  radial-gradient(820px 480px at 96% -2%, rgba(240,147,251,.08), transparent 55%),
  radial-gradient(700px 600px at 50% 120%, rgba(0,229,232,.06), transparent 60%),
  var(--bg);
```

## 2. Glow gradients & semantic mapping

Four luminous gradients carry all meaning. **Tier** = which glow; **staleness** = a heat scale that overrides tier color when a project is at risk.

| Glow | Gradient | Solid | Meaning |
|---|---|---|---|
| **Teal** | `linear-gradient(135deg,#00A3A6,#00E5E8)` | `#00E5E8` | tier **active**; staleness **fresh** (<14d) |
| **Blue** | `linear-gradient(135deg,#00F2FE,#4FACFE)` | `#4FACFE` | tier **stable** (shipped; staleness-exempt) |
| **Purple** | `linear-gradient(135deg,#F093FB,#F5576C)` | `#F093FB` | tier **dormant** |
| **Slate** | `linear-gradient(135deg,#64748B,#94A3B8)` | `#94A3B8` | tier **archive** |

**Staleness heat scale** (the warning colors, used on the activity bars and card left-edges):

| State | Color | Rule |
|---|---|---|
| fresh | `#00E5E8` teal | active <14d |
| stale | `#FBBF24` amber | active 14–29d |
| very-stale / decide | `#FF4D6D` red | active ≥30d (→ `/ship` if shipped · `dormant/` if blocked) |

These mirror the workspace lifecycle rule (`CLAUDE.md` §1/§7). The tiers carried in the
generator are `TIERS = [(name, solid, gradient), …]`; staleness solids are `FRESH/STALE/VSTALE`.

## 3. Typography (vendored, offline)

Three families, `@font-face`'d from `vendor/*.woff2` (no Google Fonts at runtime):

| Role | Family | Files | Used for |
|---|---|---|---|
| Display | **Outfit** | `outfit-600/700.woff2` | `h1` (gradient-clipped), tier headers |
| Body | **Inter** | `inter-400/600.woff2` | card text, paragraphs, Chart.js labels |
| Mono | **Fira Code** | `firacode-400.woff2` | metadata, stack chips, dates, captions, threshold labels |

```css
--display:'Outfit',ui-sans-serif,system-ui,sans-serif;
--sans:'Inter',ui-sans-serif,system-ui,-apple-system,"Segoe UI",Roboto,sans-serif;
--mono:'Fira Code',ui-monospace,"SF Mono",Menlo,Consolas,monospace;
```

> Re-vendor a weight: `curl -fsSL -o vendor/<file>.woff2 https://cdn.jsdelivr.net/npm/@fontsource/<family>@5/files/<family>-latin-<weight>-normal.woff2`, then add an `@font-face`. Keep the system fallbacks so a missing file degrades gracefully.

## 4. Glassmorphism & components

The signature: semi-transparent surfaces with `backdrop-filter: blur(16px)` over the ambient glows.

```css
.glass /* panels & cards */ {
  background: rgba(17,24,43,0.7);
  border: 1px solid rgba(255,255,255,0.08);
  border-radius: 16px;
  backdrop-filter: blur(16px); -webkit-backdrop-filter: blur(16px);
  box-shadow: 0 8px 32px rgba(0,0,0,0.2);
}
```

- **`header`** — gradient-clipped `h1` (`☀ gravity · mission control`, Outfit, blue→purple text-fill gradient) + mono "N projects · generated DATE" meta.
- **`#stars`** — a fixed, full-viewport `<canvas>` starfield at `z-index:-1` (click-through), drawn behind every glass surface; opacity from the theme's `--stars`, star color from `--chart-axis`.
- **`#tip`** — the orbital-map planet tooltip: a fixed glass card (name / tier · stack · Nd ago / focus line) positioned next to the hovered planet, clamped to the viewport. All text HTML-escaped in JS (focus lines carry literal `<angle>` brackets).
- **`.panel.glass`** — the two instrument containers (orbital map + staleness bars); uppercase mono `h3` label.
- **`.tier`** — a tier group. `h2` (Outfit, uppercase) is preceded by a glowing gradient chip (`::before`, `background:var(--grad)` + `box-shadow:0 0 12px -1px var(--accent)`); `.tcount` is the muted mono count. Section carries `--accent` (solid) + `--grad` (gradient) inline.
- **`.card.glass`** — one per project. A 3px left bar (`::before`, `background:var(--bar)`, default `var(--grad)`; overridden to `#FBBF24` on `.stale`, `#FF4D6D` on `.vstale`). Head = `🚨/⚠` marker + bold name + mono "Nd ago"; then a mono `.stack` chip and the `.focus` line.
- **`.cap`** — mono caption under the staleness chart (the legend for colors + threshold lines).
- **`footer`** — muted mono provenance line.

**Hover (cards):** lift + focus-glow.
```css
.card:hover {
  transform: translateY(-5px);
  background: var(--surface-hover);
  border-color: var(--border-focus);
  box-shadow: 0 12px 40px rgba(0,242,254,0.15);
}
```

## 5. Instruments (orbital map + Chart.js bar, vendored)

Two panels side by side: the SVG **orbital map** (left, `minmax(300px,460px)`) and the Chart.js staleness bar (right). Chart chrome reads CSS vars (`--chart-axis`, `--chart-grid`).

- **Orbital map — "tiers as rings"** (replaced the old doughnut) — an SVG star system built by JS from the inline payload (`DATA.orbit`), `viewBox 460×330`:
  - **The sun = gravity itself**: a warm radial-gradient halo + breathing gold core (`#FFE9A8`), mono `GRAVITY` label. The one warm constant across themes — it's the workspace, not a status.
  - **Rings = tiers**, inner → outer: `active` (r 64) · `stable` (r 96, solid — the literal *stable orbit*) · `dormant` (r 123) · `archive` (r 146, dashed + dimmed — the graveyard orbit). Ring stroke = `--chart-grid`; mono uppercase label above each ring's top.
  - **Planets = projects**, evenly spaced per ring (active planets r 6.5, others r 5). **Fill is the same semantics as the bars**: active = staleness heat (teal/amber/red); stable/dormant/archive = tier solid (their quiet is intentional — never heat). An **active** planet past the 30d line **pulses red** (`redpulse`) — visibly "drifting outward"; stable planets never pulse.
  - **Kepler drift**: each ring's planet group rotates continuously, inner faster (0.032 → 0.007 deg/frame). Drift **pauses while a tooltip is open** and is disabled under reduced motion.
  - **Hover → `#tip`** (see §4); **click → scrolls to the project's card** (`id="card-<slug>"`, slug from `slugify()` in the generator — same function feeds the payload, so planet↔card links can't drift) and flashes it (`.card.flash`, 1.6s).
  - Legend under the map (`#olegend`) = tier dots + counts from `DATA.tiersMeta` — the doughnut's job, one line.
- **Bar — "Staleness · active"** — horizontal (`indexAxis:'y'`), one bar per active project, **sorted stalest-first**. Bar color = the staleness heat scale. `stable`/`dormant`/`archive` are **excluded** (their age is intentional — charting them would be a false alarm).
  - **Threshold "move lines"** are drawn by a tiny inline plugin `thresholdLines` (no dependency): dashed verticals at **14d** (`#FBBF24`, "14d stale") and **30d** (`#FF4D6D`, "30d → move"), labeled at the top. `scales.x.suggestedMax` is forced ≥34 so both lines stay visible even when every bar is fresh; `layout.padding.top:14` leaves room for the labels. A bar crossing the red line is the visual "move it" warning.

## 6. Micro-animations

- **`fadeIn`** — `opacity 0→1` + `translateY(10px→0)`, on header, charts, and each tier section.
- **Card hover-lift** — `transform: translateY(-5px)` with `cubic-bezier(.4,0,.2,1)` over 0.3s.
- **Starfield twinkle** — each canvas star's alpha oscillates on its own sine phase/speed (rAF loop shared with the orbit drift). Under reduced motion the field is drawn **once, statically** at alpha .55.
- **Orbital drift** — the rings' planet groups rotate (inner faster); paused while hovering a planet so the tooltip target holds still.
- **`sunbreathe`** — the gravity core scales 1→1.12 over 5s. **`redpulse`** — 30d+ planets fade 1→.4 over 1.8s. Both are semantic motion (alive vs at-risk), not decoration.
- **Traveling border light (chart panels only)** — on `.panel:hover`, a dim **teal→purple** comet sweeps continuously around the border. A `conic-gradient(from var(--bd-angle), …)` on `.panel::after`, masked to a 1.5px ring (`mask-composite:exclude`) so the interior stays transparent and click-through; the angle animates via the registered `@property --bd-angle` (`@keyframes bd-spin`, 3.4s linear), fading in at `opacity:.6`. Deliberately scoped to the two chart panels, not the project cards. (`@property` needs modern Chromium/Edge/Safari 16.4+ — degrades to a static dim ring elsewhere.)
- **Accessibility off-switch** — `@media (prefers-reduced-motion: reduce)` disables all animations/transitions (the border light falls back to a static dim ring on hover).

## 7. Extending the dashboard (recipes)

- **Change the palette** — edit `:root` + the `TIERS` / `FRESH/STALE/VSTALE` constants in the generator together; rerun.
- **Add/rename a tier** — add a `(name, solid, gradient)` row to `TIERS` (keep a glow that carries meaning) and a matching `## name/` section in `PROJECTS.md`.
- **Adjust the move-lines** — edit the `THRESHOLDS` array in the chart JS (values, colors, labels). Keep them aligned with the lifecycle rule in workspace `CLAUDE.md`.
- **Add a font weight** — vendor the woff2 (see §3) + add an `@font-face`.
- Always finish by regenerating and confirming: new palette present, `0` unreplaced tokens, charts render.
